# EffiNote 产品需求文档 (PRD)

## 1. 产品愿景

**EffiNote** 是一款极简的 Android 习惯管理工具。它通过“**目标拆解**”和“**零路径打卡**”的设计理念，消除用户在养成习惯时的心理压力和操作负担。

---

## 2. 核心概念：阶段式量化打卡

与传统的“完成/未完成”二元逻辑不同，EffiNote 引入了 **“进度锚点”** 概念。

* **总量与单位：** 用户为每个事项设定一个具体的数值（如 2000ml 水、50 个俯卧撑、1 小时阅读）。
* **阶段拆解：** 用户设置将总量分为 $n$ 个阶段。系统自动计算每个阶段需要达到的数值。
* **线性解锁：** 任务必须按阶段顺序完成，这种“闯关式”的体验能增强打卡的获得感。

---

## 3. 功能模块说明

### 3.1 任务创建与管理

* **创建引导：**
* **任务命名：** 输入简洁的任务名称。
* **频率设定：** 支持每日循环、每周循环、每月循环。
* **量化目标：** 输入“目标数值”并从预设单位（个、次、组、分钟、米、毫升等）中选择或自定义。
* **阶段划分：** 用户拖动滑块或输入数字决定拆分数。系统即时反馈每阶段的增量。


* **任务列表：**
* 以卡片形式展示所有任务。
* 显示当前总进度（百分比/进度条）及当前所处的阶段说明。



### 3.2 桌面组件 (Widget) —— 核心交互区

这是 EffiNote 的灵魂，旨在让用户无需打开 App 即可完成 90% 的操作。

* **多任务卡片：** 标准 Widget 支持同屏显示 3 条高频任务。
* **动态交互按钮：**
* 按钮上动态显示“当前阶段需完成的量”（例如：当前需做 25 个，按钮文案即为“+25”）。
* 点击按钮后，按钮状态发生物理感知的反馈（如颜色填充、缩放动画）。


* **进度可视化：**
* 使用极简的线性进度条或分段进度条。
* 显示“已完成阶段 / 总阶段”的文字标识。


* **自动重置：** 每日、周、月根据预设频率，在桌面端自动归零并更新显示内容。

## 4. 用户交互路径 (User Journey)

1. **初次设置：** 打开 App -> 创建“每日饮水” -> 设置总量“2000ml” -> 设置阶段“4 段” -> 保存。
2. **添加组件：** 返回手机桌面 -> 长按添加 EffiNote Widget。
3. **日常打卡：** * 用户喝完一杯水（500ml） -> 点击 Widget 上的“+500”按钮。
* 确认弹窗（可选）：Widget 弹出微型确认气泡。
* **即时反馈：** 进度条增长 25%，按钮文字变为下一阶段目标，原位置显示已达成勾选。


4. **成就查看：** 偶尔进入 App 查看历史连续达标天数。

---

## 5. 视觉与设计规范 (Design Language)

### 5.1 视觉风格

* **设计语言：** 遵循 **Material You (Material 3)** 规范。
* **色彩：** 支持动态配色，Widget 背景色根据系统壁纸自动适配，保持桌面和谐。
* **排版：** 大量留白，字体清晰，强调“呼吸感”。

### 5.2 状态表达

| 状态 | 表现形式 |
| --- | --- |
| **未开始** | 进度条留白，按钮处于高亮可点击状态。 |
| **进行中** | 进度条部分填充，显示当前阶段目标。 |
| **已完成** | 进度条全满，条目整体半透明，显示“已达成”图标。 |
| **跨天逾期** | 进度条重置，根据历史数据记录失败，开始新一轮。 |

---

## 6. 未来扩展性考虑

* **多尺寸适配：** 提供 1x1（单任务大图标）、2x2（双任务）、4x2（多任务列表）等多种 Widget 尺寸。
* **快捷快捷指令：** 与 Android 快捷方式 (Shortcuts) 集成，支持通知栏快速打卡。
* **统计图表：** App 内部增加周/月维度的完成率趋势图。
